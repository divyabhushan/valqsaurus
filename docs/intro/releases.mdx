## V2.0.4 (Mar 30 , 2021)

#### New Features

* <Link to={useBaseUrl('storage/admin-console#valq-sign-in-methods')}>ValQ Sign-in Methods</Link>: ValQ allows 2-way sign-in for the users. Users can log-in with Native username/password managed by ValQ apart from Microsoft Office 365 sign-in.

#### Enhancements

* Lengthy <Link to={useBaseUrl('intro/node#1-node-components')}>Node name</Link> is truncated in the Tree view to avoid overlapping on other values.

#### Bug Fixes

* The <Link to={useBaseUrl('model/node#quick-editor-in-node-tab')}>Quick Editor</Link> does not show **Link to Model** drop-down option in the **Node Type** column.
* <Link to={useBaseUrl('simulation/tableview#1-search-for-a-node')}>Node search</Link> is fixed in the Table view.
* The node level and tool tip both display 01, when you use the **Export to Excel** and **Import from Excel** features in <Link to={useBaseUrl('model/node#quick-editor-in-node-tab')}>Quick Editor</Link>.
* Empty or 0 <Link to={useBaseUrl('model/node#7-secondary-kpis')}>Secondary KPI</Link> values do not show error.
* Fixed the Node Display in tree view. When you select any <Link to={useBaseUrl('simulation/navigation_panel#4-value-display')}>Value Display</Link> scale ( 0, 0k, 0.0m, 0.0b ) in the Navigation panel field, the value of the node does not overlaps.
<!-- * The node configuration pop-up of a merged node is fixed. -->
* Secondary KPI 0 values do not disappear when you switch on to a sub model or model merged node.
* Power BI filters interact with ValQ for Node value and Series switcher properly
* The <Link to={useBaseUrl('storage/history-logs#planning-series-logs')}>View Allocation History</Link> is grayed out when we get the confirmation popup for **Reset All** option.
* <Link to={useBaseUrl('model/node#iii-value-prefix-and-value-suffix')}>Prefix and suffix</Link> gets displayed now even when the value of the node is 0.
* In the <Link to={useBaseUrl('model/node#25-value-range')}>Node Configuration > Value Range</Link> min and max fields and <Link to={useBaseUrl('model/settings#5-conditional-formatting')}>Model > Settings > Conditional Formatting</Link> threshold sections, you can now enter 1/100th value and lesser.
* All the leaf nodes are exported in the <Link to={useBaseUrl('storage/export')}>Export as Datasource</Link> Tree structure, when the option **Export only leaf nodes** is selected.
* If a <Link to={useBaseUrl('storage/user-security')}>model is shared</Link>, secured nodes and sub models will not be visible for other user(s) upon searching across all the screens.
* When the simulation months is set as Jan to Dec then the tooltip of YtD in the <Link to={useBaseUrl('simulation/navigation_panel#4-value-display')}>Value Display</Link> section shows the three letter abbreviation that you set in the <Link to={useBaseUrl('model/data-series#2-time-aggregation-labels')}>Time aggregation Labels</Link> section.

## V2.0.3

#### New Features

* Writeback Scenario feature is also available in the <Link to={useBaseUrl('simulation/navigation_panel#1-scenarios')}>Simulate > Compare Scenarios</Link> screen.

#### Enhancements

* In a <Link to={useBaseUrl('planning/plan_features#31-allocate-children')}>Top-Down allocation</Link> for subtraction calculation method, the change in the top node's value is distributed proportionally.
* In a Simple Dynamic Model, the <Link to={useBaseUrl('model/node#4-simulation')}>Simulation is enabled only for leaf nodes</Link> by default.
* The <Link to={useBaseUrl('model/node#iii-value-prefix-and-value-suffix')}>Prefix and Suffix</Link> node values have a space character added to the text for clarity.

#### Bug Fixes

* Changes made to <Link to={useBaseUrl('storage/model-merge')}>Secondary KPIs in linked model</Link> now reflect in the main model.
* You can edit to <Link to={useBaseUrl('simulation/navigation_panel#1-scenarios')}>rename a scenario</Link> even for the model that is not uploaded to the workspace yet.
* In the <Link to={useBaseUrl('planning/plan_features#35-weight')}>Plan > **Spread by Weights**</Link> feature you can delete the node name (**Choose Node** field) with the Backspace keystroke.

## V2.0.2

#### New Features

* Node limit has been lifted on all the plans. Now models can be built for unlimited nodes.
* Upgrade pop-up has been added to the visual on PBI desktop and Service for non-enterprise plans.

## V2.0.0 (Jan 20 , 2021)

#### New Features

* <Link to={useBaseUrl('storage/user-security')}>User Security</Link> feature in ValQ Collaborate to grant specific access to the Model/nodes, Scenarios, and Planning Series to user(s) and group(s).
* <Link to={useBaseUrl('analysis/sensitivity-analysis')}>Sensitivity Analysis</Link> of the effect of varied input KPIs on a selected target node. You can select the input nodes that are to be simulated and percentage of variance. Together the 2 parameters determine the result on the output node in the form of a Tornado chart.
* Digital Board room experience with <Link to={useBaseUrl('simulation/touch-support')}>Touch support</Link>. ValQ features are now automatically optimized for Touch Support on touch-enabled devices for both Canvas and Tree view.
* The **Plan** tab includes <Link to={useBaseUrl('planning/plan_features#7-writeback')}>Writeback</Link> feature that triggers to send data to an external API.
* You can enable to show the <Link to={useBaseUrl('model/node#show-secondary-kpis-in-table-view')}>Secondary KPIs </Link> in the Table view.

#### Enhancements

* Planning series implements single lock icon as a toggle to <Link to={useBaseUrl('planning/plan_features#33-hold-and-release')}>Hold/Release</Link> the cells.
* Plan/Simulation/Variance Analysis tab introduces <Link to={useBaseUrl('planning/plan_features#6-size-columns-to-fit')}>Size columns to fit</Link> option to auto-size the columns width.
* Added <Link to={useBaseUrl('simulation/navigation_panel#show-composite-nodes')}>Show Composite Nodes</Link> as a toggle in Navigation Panel for models which have Composite Nodes.
* The <Link to={useBaseUrl('model/node#manage-columns-settings')}>Manage Columns</Link> settings in the **Model > Node > Quick Editor** is grouped similar to the labels in the **Edit Node** configurations.
* The <Link to={useBaseUrl('storage/export')}>Export as datasource</Link> feature includes **Export all** option. This option allows you to export all the saved series/scenarios in the model including future updates.

#### Bug Fixes

* The selections made in the <Link to={useBaseUrl('analysis/variance_analysis')}>Variance Analysis</Link> tab are now saved and retained even when you quit the Analyze tab.
* The value of Node completely gets diplayed in the <Link to={useBaseUrl('planning/plan_overview')}>Plan Grid (Decimal points)</Link> irrespective of the node's configuration. 
* Fixed the "Tab" button functionality in the Planning <Link to={useBaseUrl('planning/plan_features#35-weight')}>Weight</Link> feature.
The use of the "Tab" button to move to the next period is disabled for the last period and the use of the "Shift + Tab" to come back to the pervious period is disabled for the 1st period.

## V1.7.0 (May 12,2020)

#### Features

* Added <Link to={useBaseUrl('storage/model-merge')}>Model Merge</Link> feature that allows users to merge ValQ models from different users within the workspace.
* Added <Link to={useBaseUrl('planning/time_series')}>Time Series Forecasting</Link> feature that allows you make a forecast based on past data.
* You can manage the Users/Groups, and Models from your <Link to={useBaseUrl('storage/admin-console')}>Member/ValQ Console</Link> account.

#### Enhancements

* Enabled **Scenario Collaboration** (ValQ - Professional, Enterprise) for Private and Shared Scenarios with view/edit access.
* Enabled **Read-only Mode** in Power BI Service, with read-only access for *Visualization (Table/Tree mode), Nav Panel, Export, Compare, and Variance Analysis* like features.
* Added **Model Details** in the workspace, along with details like Series/Scenarios under *Member/ValQ Management Console*.
